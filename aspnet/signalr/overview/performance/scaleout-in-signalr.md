## <a name="implementation"></a><span data-ttu-id="a3ae2-140">Implementação</span><span class="sxs-lookup"><span data-stu-id="a3ae2-140">Implementation</span></span>

<span data-ttu-id="a3ae2-145">Cada instância de servidor se conecta ao backplane por meio do barramento.</span><span class="sxs-lookup"><span data-stu-id="a3ae2-145">Each server instance connects to the backplane through the bus.</span></span> <span data-ttu-id="a3ae2-146">Quando uma mensagem é enviada, ela vai para o backplane e o backplane envia para cada servidor.</span><span class="sxs-lookup"><span data-stu-id="a3ae2-146">When a message is sent, it goes to the backplane, and the backplane sends it to every server.</span></span> <span data-ttu-id="a3ae2-147">Quando um servidor recebe uma mensagem do backplane, ele coloca a mensagem em seu cache local.</span><span class="sxs-lookup"><span data-stu-id="a3ae2-147">When a server gets a message from the backplane, it puts the message in its local cache.</span></span> <span data-ttu-id="a3ae2-148">O servidor, em seguida, entrega mensagens para os clientes de seu cache local.</span><span class="sxs-lookup"><span data-stu-id="a3ae2-148">The server then delivers messages to clients from its local cache.</span></span>

<span data-ttu-id="a3ae2-149">Para cada conexão de cliente, o progresso do cliente no fluxo de mensagens de leitura é controlado usando um cursor.</span><span class="sxs-lookup"><span data-stu-id="a3ae2-149">For each client connection, the client's progress in reading the message stream is tracked using a cursor.</span></span> <span data-ttu-id="a3ae2-150">(Um cursor representa uma posição no fluxo de mensagens.) ## <a name="limitations"></a><span data-ttu-id="a3ae2-155">Limitações</span><span class="sxs-lookup"><span data-stu-id="a3ae2-155">Limitations</span></span>

<span data-ttu-id="a3ae2-156">Usando um backplane, a taxa de transferência máxima de mensagens é menor do que quando os clientes falam diretamente a um nó de servidor único.</span><span class="sxs-lookup"><span data-stu-id="a3ae2-156">Using a backplane, the maximum message throughput is lower than it is when clients talk directly to a single server node.</span></span> <span data-ttu-id="a3ae2-157">Isso ocorre porque o backplane encaminha todas as mensagens para todos os nós, para que o backplane pode se tornar um gargalo.</span><span class="sxs-lookup"><span data-stu-id="a3ae2-157">That's because the backplane forwards every message to every node, so the backplane can become a bottleneck.</span></span> <span data-ttu-id="a3ae2-158">Se essa limitação é um problema depende do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="a3ae2-158">Whether this limitation is a problem depends on the application.</span></span> <span data-ttu-id="a3ae2-159">Por exemplo, aqui estão alguns cenários típicos do SignalR:</span><span class="sxs-lookup"><span data-stu-id="a3ae2-159">For example, here are some typical SignalR scenarios:</span></span>

- <span data-ttu-id="a3ae2-160">[Servidor de transmissão](../getting-started/tutorial-server-broadcast-with-signalr.md) (por exemplo, bolsa): Backplanes funcionam bem para este cenário, porque o servidor controla a taxa na qual as mensagens são enviadas.</span><span class="sxs-lookup"><span data-stu-id="a3ae2-160">[Server broadcast](../getting-started/tutorial-server-broadcast-with-signalr.md) (e.g., stock ticker): Backplanes work well for this scenario, because the server controls the rate at which messages are sent.</span></span>
- <span data-ttu-id="a3ae2-161">[Cliente-para-cliente](../getting-started/tutorial-getting-started-with-signalr.md) (por exemplo, bate-papo): nesse cenário, o backplane pode ser um gargalo se o número de mensagens pode ser dimensionado com o número de clientes; ou seja, se a taxa de mensagens aumenta proporcionalmente de mais clientes unir.</span><span class="sxs-lookup"><span data-stu-id="a3ae2-161">[Client-to-client](../getting-started/tutorial-getting-started-with-signalr.md) (e.g., chat): In this scenario, the backplane might be a bottleneck if the number of messages scales with the number of clients; that is, if the rate of messages grows proportionally as more clients join.</span></span>
- <span data-ttu-id="a3ae2-162">[Em tempo real de alta frequência](../getting-started/tutorial-high-frequency-realtime-with-signalr.md) (por exemplo, jogos em tempo real): um backplane não é recomendado para este cenário.</span><span class="sxs-lookup"><span data-stu-id="a3ae2-162">[High-frequency realtime](../getting-started/tutorial-high-frequency-realtime-with-signalr.md) (e.g., real-time games): A backplane is not recommended for this scenario.</span></span>
