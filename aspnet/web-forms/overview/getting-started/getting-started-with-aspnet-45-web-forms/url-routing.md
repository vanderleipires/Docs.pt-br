### <a name="url-patterns"></a><span data-ttu-id="1077f-131">Padrões de URL</span><span class="sxs-lookup"><span data-stu-id="1077f-131">URL Patterns</span></span>

<span data-ttu-id="1077f-134">Quando é feita uma solicitação para seu aplicativo web, a URL é analisada em segmentos e os espaços reservados e os valores de variáveis são fornecidos para o manipulador de solicitação.</span><span class="sxs-lookup"><span data-stu-id="1077f-134">When a request to your web application is made, the URL is parsed into segments and placeholders, and the variable values are provided to the request handler.</span></span> <span data-ttu-id="1077f-135">Esse processo é semelhante à forma como os dados em uma cadeia de caracteres de consulta são analisados e passados para o manipulador de solicitação.</span><span class="sxs-lookup"><span data-stu-id="1077f-135">This process is similar to the way the data in a query string is parsed and passed to the request handler.</span></span> <span data-ttu-id="1077f-136">Em ambos os casos, informações sobre a variável está incluído na URL e passado para o manipulador na forma de pares chave-valor.</span><span class="sxs-lookup"><span data-stu-id="1077f-136">In both cases, variable information is included in the URL and passed to the handler in the form of key-value pairs.</span></span> <span data-ttu-id="1077f-137">Para cadeias de caracteres de consulta, as chaves e os valores estão na URL.</span><span class="sxs-lookup"><span data-stu-id="1077f-137">For query strings, both the keys and the values are in the URL.</span></span> <span data-ttu-id="1077f-138">Para rotas, as chaves são os nomes de espaço reservado definidos no padrão de URL e apenas os valores estão na URL.</span><span class="sxs-lookup"><span data-stu-id="1077f-138">For routes, the keys are the placeholder names defined in the URL pattern, and only the values are in the URL.</span></span>
